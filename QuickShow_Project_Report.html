Project Overview

3.1Description

QuickShow is designed to deliver an online entertainment booking experience. It allows users to browse movies, choose showtimes, select seats, and complete bookings with secure payment processing. The application also includes an admin section for show management and booking oversight.

3.2Technology Stack

  • Frontend: React, Vite, Tailwind CSS, Clerk
  • Backend: Express, Node.js, MongoDB, Mongoose
  • Authentication: Clerk
  • Payments: Razorpay
  • Background processing: Inngest

3.3Key Features

  • User sign-up and login
  • Movie browsing with cards and trailers
  • Interactive seat selection and booking
  • Secure checkout and payment confirmation
  • User booking history and favorites
  • Admin dashboard for shows and bookings

Project Structure

4.1Root Directory

The project root contains the main `README.md`, run scripts, deployment configuration, and a root `package.json` that orchestrates the client and server.

4.2Client Directory

The `client` folder contains the React application. Important files include:

  • `client/src/main.jsx` – application bootstrap and Clerk setup
  • `client/src/App.jsx` – routing and layout
  • `client/src/auth/ProtectedRoute.jsx` – authentication guard
  • `client/src/context/AppContext.jsx` – global state, API client, and auth token handling
  • `client/src/pages/SeatLayout.jsx` – seat selection and booking flow

4.3Server Directory

The `server` folder contains Express APIs, middleware, models, and background job hooks.

  • `server/server.js` – server initialization and route mounting
  • `server/controllers/bookingController.js` – booking and payment logic
  • `server/routes/*` – API route definitions for shows, bookings, users, and admin
  • `server/inngest/index.js` – asynchronous functions for payment checks and email notifications

Booking Flow

6.1Seat Selection

The `SeatLayout` page displays theater rows and seat buttons. Seats can be selected up to a maximum of five, and occupied seats are disabled.

6.2Booking Creation

The frontend creates a booking with `/api/booking/create`. If Razorpay is unavailable during development, the app uses a fallback mock flow to simulate booking and payment.

const { data } = await axios.post(
  "/api/booking/create",
  { showId: selectedTime.showId, selectedSeats },
  { headers: { Authorization: `Bearer ${await getToken()}` } }
);

if (!data.success) {
  return toast.error(data.message || "Unable to create booking.");
}

6.3Payment Verification

After checkout, the client verifies the payment with `/api/booking/verify`, and the backend marks the booking as paid and finalizes seat occupancy.

Server Architecture

7.1Express Setup

The server uses Express, CORS middleware, Clerk middleware, and mounts API routes for booking, show data, user actions, and admin management.

import express from "express";
import cors from "cors";
import { clerkMiddleware } from "@clerk/express";

const app = express();
app.use(express.json());
app.use(cors());
app.use("/dev", devRouter);
app.use(clerkMiddleware());
app.use("/api/booking", bookingRouter);
app.use("/api/show", showRouter);

7.2Booking Controller

The controller protects seat reservation with atomic updates and uses expiration times so held seats are released if payment is not completed.

const show = await prepareShowAndCleanup(showId);
if (!show) return res.json({ success: false, message: "Show not found." });

const booking = await Booking.create({ user: userId, show: showId, amount, bookedSeats: selectedSeats });

const holdExpiry = new Date(Date.now() + 10 * 60 * 1000);
const seatUpdates = {};
selectedSeats.forEach((s) => {
  seatUpdates[`occupiedSeats.${s}`] = { bookingId: booking._id.toString(), expiresAt: holdExpiry };
});

Authentication and Authorization

8.1Clerk Integration

Clerk manages authentication on the frontend and backend. The application checks whether Clerk is configured and uses the Clerk React components for sign-in and sign-up.

8.2Admin Authorization

The backend protects admin routes by checking the authenticated Clerk user and confirming admin privileges through the server middleware.

Setup and Run Instructions

10.1Prerequisites

  • Node.js
  • npm
  • MongoDB or MongoDB Atlas

10.2Installation

Install dependencies for both client and server:

cd server
npm install
cd ../client
npm install

10.3Configuration

Create `.env` files from the provided examples in `server/.env.example` and `client/.env.example` and fill in Clerk and payment credentials.

10.4Run the Application

cd server
node server.js
cd ../client
npx vite --host

Open http://localhost:5173 in the browser.
